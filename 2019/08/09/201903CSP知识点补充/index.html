好好刷题！

需要补充的知识点有：栈的应用（表达式求值），sscanf和sprintf的用法，scanf,gets,fget,cin,getline的比较。

栈的应用（表达式求值）

  • 栈既可应用于中缀表达式求值，也可应用于后缀表达式求值。

    后缀表达式求值

  • 后缀表达式求值是计算机比较喜闻乐见的，它的特点是不需要括号就能表示出整个表达式哪部分运算先进行，也就是说不需要考虑优先级，这非常符合计算机的处理方式。只需要一个操作数栈就可以对后缀表达式进行求值了。
  • 比如计算3+5*(2-4),可以先得到后缀表达式为3524-*+。具体在计算时先将所有操作数入栈，遇到操作符之后取两个操作数出栈，计算得到的结果入栈，一直这样直到计算完。
  • 比较麻烦的是一般给我们的都是中缀表达式，需要将中缀表达式转换为后缀表达式。中缀表达式转换为后缀表达式的时候需要用到栈来辅助存储，具体过程如下：

    1. 如果遇到操作数，直接将其输出。
    2. 如果遇到操作符，则将其放入到栈中，遇到左括号时也将其放入栈中。
    3. 如果遇到一个右括号，则将栈元素弹出，将弹出的操作符输出直到遇到左括号为止。注意，左括号只弹出并不输出。
    4. 如果遇到任何其他的操作符，如（“+”， “*”，“（”）等，从栈中弹出元素直到遇到发现更低优先级的元素(或者栈为空)为止。弹出完这些元素后，才将遇到的操作符压入到栈中。有一点需要注意，只有在遇到" ) "的情况下我们才弹出" ( "，其他情况我们都不会弹出" ( "。
    5. 如果读到了输入的末尾，则将栈中所有元素依次弹出。

    中缀表达式求值

  • 中缀表达式就是我们最常见到的表达式的形式。中缀表达式求值，也就是直接对输入的表达式进行求值。为简单起见，这里假设只涉及加减乘除和小括号，并且操作数都是正整数，不涉及更加复杂的数据或运算。
  • 计算思路为：

    1. 使用两个栈，stack0用于存储操作数，stack1用于存储操作符。
    2. 从左往右扫描，遇到操作数入栈stack0。
    3. 遇到操作符时，如果优先级低于或等于栈顶操作符优先级，则从stack0弹出两个元素进行计算，并将结果压入stack0，继续与栈顶操作符的比较优先级。
    4. 如果遇到操作符高于栈顶操作符优先级，则直接入栈stack1。
    5. 遇到左括号，直接入栈stack1，遇到右括号，则直接出栈并计算，直到遇到左括号。

    第二题思路

  • 不用考虑中缀表达式求值思路的第5条然后去做就ok。

    sscanf和sprintf的用法

  • sscanf从单词上可以理解为string+scanf, sprintf则可以理解为string+printf,他们都在stdio.h头文件下。

    sscanf的应用

    sscanf与scanf类似，都是用于输入的，只是后者以屏幕(stdin)为输入源，前者以固定字符串为输入源。sscanf函数原型为int sscanf(const char *str, const char *format, ...).将参数str的字符串根据参数format字符串来转换并格式化数据，转换后的结果存于对应的参数内。具体功能如下：

    1. 根据格式从字符串中提取数据。如从字符串中取出整数、浮点数和字符串等。
    2. 取指定长度的字符串。
    3. 取到指定字符为止的字符串。
    4. 取仅包含指定字符集的字符串。
    5. 取到指定字符集为止的字符串。
    6. 将十六进制或八进制的字符串转换为整型变量。

    • `*`亦可用于格式中, 即`%*d`和`%*s`加了星号`*`表示跳过此数据不读入. (也就是不把此数据读入参数中)。

scanf,gets,fget,cin,cin.getline，getline的比较

  • scanf和cin遇到非空白字符结束，gets()和fgets()在读取时遇到换行符或EOF时停止，由于gets()不检查目标数组是否能容纳输入，所以很不安全，已被C11标准废弃。cin.getline会一次读取多个字符(包括空白字符）。它以指定的地址为存放第一个读取的字符的位置，依次向后存放读取的字符，直到读满N-1个，或者遇到指定的结束符为止。若不指定结束符，则默认结束符为’\0’。其语法为其语法为:cin.getline(字符指针(char*),字符个数N(int),结束符(char));。getline接收一个字符串，可以接收空格并输出，需包含头文件#include<cstring>。一般在对较长的字符串输入时，选择fegts函数比较快。
